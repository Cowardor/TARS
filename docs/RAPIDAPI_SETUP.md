# Настройка RapidAPI в n8n (Пошаговая инструкция)

**Дата:** 2026-01-19
**Цель:** Исправить ошибку RapidAPI в workflow

---

## ⚠️ Частая проблема: "Node does not have any credentials set"

Эта ошибка означает, что RapidAPI ключ не настроен правильно в n8n.

---

## 🔧 Пошаговое решение

### Шаг 1: Получить RapidAPI ключ

1. Откройте: https://rapidapi.com/apidojo/api/aliexpress-datahub
2. **Войдите** или **зарегистрируйтесь** (можно через Google)
3. Нажмите **"Subscribe to Test"**
4. Выберите **"Basic Plan"** - **$0/месяц**
5. Нажмите **"Subscribe"**
6. Вернитесь на страницу API
7. Справа в блоке **"Code Snippets"** найдите:
   ```
   X-RapidAPI-Key: ВАШ_КЛЮЧ_ЗДЕСЬ
   ```
8. **Скопируйте этот ключ** (примерно 50 символов)

**Пример ключа:**
```
1234567890abcdefghijklmnopqrstuvwxyz1234567890ab
```

---

### Шаг 2: Создать Credential в n8n

1. Откройте n8n: http://localhost:5678
2. В левом меню нажмите **"Credentials"** (иконка ключа 🔑)
3. Нажмите **"+ Add Credential"** (справа вверху)
4. В поиске введите **"Header Auth"**
5. Выберите **"Header Auth"**

**Заполните форму:**

| Поле | Значение |
|------|----------|
| **Credential Name** | `RapidAPI Key` |
| **Name** | `X-RapidAPI-Key` |
| **Value** | *Вставьте ваш ключ* |

6. Нажмите **"Save"**

---

### Шаг 3: Импортировать тестовый workflow

1. В n8n нажмите **"+" (New workflow)**
2. Нажмите **"Import from File"** (или Ctrl+O)
3. Выберите файл: `c:\Users\illia\Documents\TARS\workflows\scout-test-v1.json`
4. Workflow появится в редакторе

**Должно быть 6 нод:**
1. Ручной запуск
2. Параметры поиска
3. RapidAPI: AliExpress поиск
4. Парсинг результатов
5. Фильтр (рейтинг + отзывы)
6. Расчет экономики

---

### Шаг 4: Подключить RapidAPI credential к ноде

1. Кликните на ноду **"RapidAPI: AliExpress поиск"** (3-я нода)
2. Справа откроется панель настроек
3. Найдите секцию **"Credentials"**
4. В поле **"Credential to connect with"** выберите **"RapidAPI Key"** (созданный в Шаге 2)
5. Нажмите **"Save"** (или просто закройте панель)

---

### Шаг 5: Протестировать RapidAPI

1. Кликните на ноду **"RapidAPI: AliExpress поиск"**
2. Нажмите **"Test step"** (или **"Execute Node"**)
3. Подождите 2-5 секунд

**✅ Ожидаемый результат:**
Внизу появится JSON ответ с данными о товарах AliExpress

**❌ Если ошибка:**
- Проверьте, что ключ скопирован правильно (без пробелов)
- Проверьте, что выбрали **Basic Plan** в RapidAPI
- Попробуйте обновить страницу n8n (F5)

---

### Шаг 6: Запустить весь workflow

1. Нажмите **"Execute Workflow"** (кнопка внизу)
2. Наблюдайте, как данные проходят через все ноды
3. В конце проверьте результаты в последней ноде **"Расчет экономики"**

**✅ Ожидаемый результат:**
Список товаров с:
- Названием
- Ценой на AliExpress
- Рейтингом и отзывами
- Расчетом прибыли
- Рекомендацией (STRONG BUY / BUY / MAYBE / PASS)

## ⚠️ Troubleshooting

### Ошибка: "Unauthorized" или "Invalid API Key"

**Причина:** Неправильный RapidAPI ключ

**Решение:**
1. Проверьте ключ на странице RapidAPI
2. Убедитесь, что скопировали весь ключ целиком
3. Пересоздайте credential в n8n

---

### Ошибка: "Rate limit exceeded"

**Причина:** Превышен лимит 100 запросов/день

**Решение:**
1. Подождите до завтра (лимит сбросится)
2. Или апгрейдьте на Pro план ($10/месяц)

---

### Ошибка: "No results found"

**Причина:** Неправильный формат ответа от API

**Решение:**
1. Проверьте в ноде **"Парсинг результатов"**
2. Посмотрите на `raw_response` в выходных данных
3. Если нужно - скорректируйте парсинг в JavaScript коде

---

### Workflow выполняется, но нет отфильтрованных товаров

**Причина:** Слишком строгие критерии фильтра

**Решение:**
В ноде **"Фильтр (рейтинг + отзывы)"** измените:
```javascript
const minRating = 4.0;  // Было 4.5
const minReviews = 50;  // Было 100
```
